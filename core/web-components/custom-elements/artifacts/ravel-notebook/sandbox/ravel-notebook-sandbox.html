ravel-notebook — Ravel Audio Architecture

Introduction

Raveling is a Web Component design system for music apps, games, and interactive tools. It communicates entirely through a lightweight pub/sub bus — no framework, no global singletons. Components look like instruments, not form fields.

This notebook walks through the three-layer architecture: gadgets (device adapters), brokers (message routers), and gidgets (interactive UI components).

The Messaging System

Every Ravel component inherits from RavelElement, which exposes two messaging primitives:

  • sendMessage(channel, cmd, content) — targeted pub/sub via RavelMessenger
  • broadcastMessage(channel, cmd, content) — fires a CustomEvent on window

Message shape is always { cmd: string, content: unknown, version: number }. Brokers are invisible DOM elements that translate between the external world (MIDI, network, sensors) and Ravel's message format.

// Subscribing to MIDI note events from ravel-midi-broker window.addEventListener('midi', (e) => { const { cmd, content } = e.detail; if (cmd === 'note-on') { const { note, velocity, channel } = content; const freq = 440 * Math.pow(2, (note - 69) / 12); osc.frequency.setValueAtTime(freq, ctx.currentTime); gain.gain.setValueAtTime(velocity / 127, ctx.currentTime); } if (cmd === 'note-off') { gain.gain.linearRampToValueAtTime(0, ctx.currentTime + 0.08); } if (cmd === 'cc' && content.controller === 1) { // mod wheel → vibrato depth lfo.gain.setValueAtTime(content.value / 127 * 50, ctx.currentTime); } });

Brokers

A broker is placed once in the page HTML. ravel-midi-broker bridges Web MIDI (via ravel-midi-gadget) into the Ravel bus. It parses raw MIDI bytes and broadcasts structured events. It also encodes outbound structured messages back to raw bytes for transmission.

The broker below is live. Its RX LED flashes when raw MIDI arrives; TX flashes when parsed events are broadcast. During MIDI clock both LEDs stay continuously lit.

<!-- Place the broker in your page HTML --> <ravel-midi-broker label="ravel-midi-broker" channel="midi"> </ravel-midi-broker> <!-- Send raw MIDI to it from ravel-midi-gadget --> <!-- RavelMessenger.sendMessage('ravel-midi-broker', 'raw', { data: [0x90, 60, 80], deviceName: 'My Keyboard' }); -->

Message Schema

Every message on the Ravel bus follows the same envelope. The broker emits structured events on the channel attribute value (default "midi"). The output below shows a typical note-on event payload.

{ "cmd": "note-on", "content": { "channel": 0, "note": 60, "velocity": 80, "deviceName": "Arturia KeyStep 37" }, "version": 1 }

What's Next

Connect a ravel-midi-gadget to a real MIDI device and the broker automatically parses incoming bytes into these structured events. Wire the "midi" channel to a ravel-fluxly-sound-engine or your own AudioWorklet for a fully live synthesis pipeline.

  • Add device="Arturia KeyStep 37" to filter by device name
  • Change channel to namespace multiple brokers on the same page
  • Use observable="events" on any component to trace all messages
