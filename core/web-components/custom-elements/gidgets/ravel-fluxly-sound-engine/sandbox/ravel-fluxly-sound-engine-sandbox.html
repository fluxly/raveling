ravel-fluxly-sound-engine

Headless AudioWorklet sample player — variable rate, forward and reverse

Browsers block audio until a user gesture. Click Resume audio inside the panel (or any other button below) to unblock the AudioContext.
show-control — debug rate panel visible
external rate control via page
no-loop — plays once then stops
headless — no show-control (invisible by default)

The element below has no show-control attribute — it renders nothing. The button sends a playback-rate message to it.

attributes & API

filename — URL of the WAV file to load (default sample-0.wav).
message-id — message channel name (default ravel-fluxly-sound-engine).
no-loop — stop playback at the end instead of looping.
show-control — show the built-in rate slider panel.

Messages received on message-id channel:
  { cmd: 'playback-rate', content: number } — set rate (±16.0; negative = reverse).
  { cmd: 'resume' } — restart playback after no-loop end.

Tip: pair with <ravel-filmstrip> — listen for play-sound events and forward them as playback-rate messages.
