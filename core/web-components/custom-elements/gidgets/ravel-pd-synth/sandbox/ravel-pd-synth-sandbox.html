ravel-pd-synth

Runs a Pure Data patch inside an AudioWorklet via Emscripten WASM

component renders — LED reflects audio state
LED status states
idle — not yet configured or waiting for first interaction
loading — fetching worklet, engine JS, and patch file
ready — AudioWorklet running, PD patch loaded
error — fetch failed, worklet error, or WASM init failure
send signals to the synth via message channels

Audio requires real engine + patch files at the configured paths.
These controls broadcast on the configured channels — the synth forwards them to PD receivers once the worklet is running.

tempo 120
volume 0.80
attributes & signal routing

engine — path prefix; loads {engine}/worklet.js and {engine}/engine.js.
patch-src — URL of the .pd patch file to fetch and pass to the worklet.
signals — comma-separated channel:receiver pairs.
  e.g. signals="tempo:tempo-recv, volume:vol-recv"

Audio initializes on the first user interaction (click or pointerdown) due to
browser autoplay policy. The LED turns orange while loading, lime when ready.

Signal messages arrive as { cmd: any, content: number }.
The content value is forwarded as a float to the named PD receiver.

The engine JS must be compiled with Emscripten SINGLE_FILE=1 (WASM base64-embedded).
The worklet must register a processor named 'pd-synth-processor'.
